01 · RMSNorm คืออะไร
Normalization — ทำไมถึงจำเป็น

ระหว่างการ forward pass ตัวเลขใน hidden state อาจ "ระเบิด" ออกไปหรือ "หด" จนหายไป Normalization ทำหน้าที่บีบตัวเลขให้อยู่ในช่วงที่ gradient ไหลได้สม่ำเสมอ ซึ่งช่วยทั้ง training stability และ convergence speed

LayerNorm (ดั้งเดิม)
คำนวณ Mean และ Variance แยกกันสองรอบ แล้ว Centering ด้วยการลบ mean ออกก่อน ถึงจะ scale — ใช้ memory bandwidth เยอะกว่า และมี operation chain ยาวกว่า
RMSNorm (Yae Style)
ข้าม Centering ไปเลย ใช้แค่ Root Mean Square ของ hidden state โดยตรง ลด operation ลงครึ่งหนึ่ง ประหยัด compute 7–64% ตาม paper ต้นฉบับ
Paper reference: Zhang & Sennrich (2019) — "Root Mean Square Layer Normalization" พิสูจน์ว่า Centering (การลบ mean) แทบไม่มีผลต่อ model quality แต่กิน compute ฟรีๆ Qwen, LLaMA, Mistral ล้วนใช้ RMSNorm แทน LayerNorm ค่ะ
02 · คณิตศาสตร์เบื้องหลัง
สูตร RMSNorm
RMS(x) = √( (1/n) · Σ xᵢ² )
ȳ = (x / RMS(x) + ε) · γ
  • ยกกำลังสอง — hidden_states.pow(2) แต่ละ element ใน vector
  • หาค่าเฉลี่ย — .mean(-1, keepdim=True) เฉลี่ยตาม hidden_size dimension
  • บวก epsilon — + variance_epsilon ป้องกัน divide-by-zero เมื่อ variance ≈ 0
  • Reciprocal sqrt — torch.rsqrt(...) คำนวณ 1/√x ใน kernel เดียว เร็วกว่า 1/torch.sqrt()
  • Scale ด้วย γ — self.weight * ... ซึ่งเป็น learnable parameter เริ่มต้นที่ ones(hidden_size)
dtype trick: แปลง input เป็น float32 ก่อนคำนวณ variance แล้ว cast กลับเป็น input_dtype ตอน return — ป้องกัน precision loss เมื่อเทรนด้วย bf16 หรือ fp16

03 · Implementation Code
YaeRMSNorm.py PyTorch + Custom Kernel 
@use_kernel_forward_from_hub("RMSNorm")
        class YaeRMSNorm(nn.Module):
            def __init__(self, hidden_size: int, eps: float = 1e-6) -> None:
                super().__init__()
                self.weight = nn.Parameter(torch.ones(hidden_size))
                self.variance_epsilon = eps

            def forward(self, hidden_states: torch.Tensor) -> torch.Tensor:
                input_dtype = hidden_states.dtype
                # cast to float32 ก่อนเสมอ — ป้องกัน precision loss ตอนคำนวณ variance
                hidden_states = hidden_states.to(torch.float32)

                # หาค่าเฉลี่ยของกำลังสอง (ข้ามขั้น mean centering ของ LayerNorm)
                variance = hidden_states.pow(2).mean(-1, keepdim=True)

                # rsqrt = 1/√(variance + ε)  ใน single fused kernel — เร็วกว่า 1/sqrt แยก
                hidden_states = hidden_states * torch.rsqrt(variance + self.variance_epsilon)

                # คูณ learnable scale γ และ cast กลับเป็น original dtype
                return self.weight * hidden_states.to(input_dtype)
@use_kernel_forward_from_hub("RMSNorm") — decorator นี้ replace forward() ด้วย fused CUDA kernel จาก HuggingFace Hub ตอน runtime ถ้า environment รองรับ fallback กลับมา PyTorch native ถ้าไม่มี GPU kernel ค่ะ
04 · ถอดรหัส Tensor Shape ใน LLM
torch.randn(B, T, C) — รูปแบบมาตรฐาน hidden state

ทุก tensor ที่ไหลผ่าน Transformer block จะมี shape 3 มิตินี้เสมอ torch.randn(1, 2, 4) อ่านได้ทันทีว่า: batch 1, ประโยค 2 token, แต่ละ token มี feature 4 ตัว

Dim 0 · B
Batch Size
จำนวนประโยค/ชุดข้อมูลที่รันพร้อมกันใน 1 forward pass — ยิ่งมาก GPU ยิ่งทำงานขนานได้เต็มที่ (แต่กิน VRAM เพิ่ม)
Dim 1 · T
Sequence Length / Tokens
จำนวน token ในประโยค — ตัว Attention ต้องคิด relationship ทุกคู่ O(T²) นี่คือเหตุที่ long context ถึงหนักมาก
Dim 2 · C
Hidden Size / Features
เวกเตอร์ความหมาย — เรียกว่า hidden_size หรือ d_model คือ "ขนาดสมอง" ของโมเดล ยิ่งกว้างยิ่งจำได้มาก
RMSNorm normalize ตาม Dim 2 (C) — .mean(-1) คือหาค่าเฉลี่ยตาม last dimension นั่นคือ normalize แต่ละ token แยกกัน ไม่ข้าม batch หรือ sequence ค่ะ

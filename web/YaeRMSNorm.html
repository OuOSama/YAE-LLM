RMSNorm คืออะไร? (และต่างจาก LayerNorm ยังไง)

RMSNorm (Root Mean Square Normalization) เป็นเทคนิคปรับสมดุล Tensor ที่พัฒนาขึ้นมาเพื่อทดแทน LayerNorm แบบดั้งเดิมในโมเดล Transformer ยุคใหม่ ช่วยลดภาระการคำนวณและเพิ่มความเร็วอย่างมหาศาล โดยตัดส่วนการหา Mean (ค่าเฉลี่ยดั้งเดิม) ออกไป

❌ LayerNorm (ดั้งเดิม)

ต้องคำนวณทั้ง Mean และ Variance มีการทำ Centering ทำให้ใช้พลังคำนวณและแรมบน GPU สองสเต็ป

✨ RMSNorm (Yae Style)

ข้ามขั้นตอน Centering ไปเลย จับยกกำลังสองแล้วหา Mean of Squares ทันที ประหยัดพลังคำนวณลงได้ 7% - 64%

Implementation Code

YaeRMSNorm.py PyTorch + Custom Kernel 
@use_kernel_forward_from_hub("RMSNorm")
class YaeRMSNorm(nn.Module):
    def __init__(self, hidden_size: int, eps: float = 1e-6) -> None:
        super().__init__()
        self.weight = nn.Parameter(torch.ones(hidden_size))
        self.variance_epsilon = eps

    def forward(self, hidden_states: torch.Tensor) -> torch.Tensor:
        input_dtype = hidden_states.dtype
        hidden_states = hidden_states.to(torch.float32)
        
        # หาค่าเฉลี่ยของกำลังสองทันทีโดยไม่ลบ Mean ดั้งเดิม
        variance = hidden_states.pow(2).mean(-1, keepdim=True)
        
        # ทำ Scaling รวดเดียวด้วย Reciprocal Square Root
        hidden_states = hidden_states * torch.rsqrt(variance + self.variance_epsilon)
        return self.weight * hidden_states.to(input_dtype)

ถอดรหัสมิติ Tensor ยอดฮิตใน LLM

เมื่อจำลองรูปทรงข้อมูลด้วยสูตร torch.randn(1, 2, 4) ผลลัพธ์ที่ได้คือวงเล็บที่ซ้อนกัน 3 ชั้นตามแกน (Batch Size, Sequence Length, Hidden Size) ดังนี้:

Dim 0

Batch Size (1)

จำนวนชุดข้อมูลหรือประโยคที่ส่งเข้าไปรันพร้อมกันใน 1 รอบการประมวลผล

Dim 1

Sequence Length / Tokens (2)

จำนวนคำในประโยคนั้นๆ (ในตัวอย่างมี 2 คำ เลยแบ่งออกเป็น 2 แถวบน-ล่าง)

Dim 2

Hidden Size / Features (4)

เวกเตอร์ตัวเลข 4 ตัวที่เป็นตัวแทนความหมายหรือฟีเจอร์ภายในระบบ AI ของคำนั้นๆ

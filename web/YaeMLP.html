SwiGLU SiLU / Swish Production-Ready Qwen3 Architecture
01 · เจาะลึกกลไก SiLU (Swish)
ทำไมจังหวะการเปิด-ปิดประตูสัญญาณใน gate_proj ถึงต้องใช้ SiLU แทนที่จะเป็น ReLU หักมุมแบบเดิมๆ?

🧠 สูตรคณิตศาสตร์เบื้องหลัง

f(x) = x · sigmoid(x)
  • Smooth & Continuous: ทรงกราฟไม่มีการหักมุมทื่อๆ ทำให้ค่าเกรเดียนท์ (Gradient) ตอนคำนวณย้อนกลับไม่ขาดหาย ช่วยให้เทรนโมเดลชั้นลึกๆ ได้นิ่งและสเถียรมาก
  • Non-monotonicity (ก้นกราฟพ่นแดนลบ): มีพื้นที่เล็กๆ ตรงก้นกราฟที่ยอมให้ค่าติดลบไหลผ่านไปเรียนรู้ต่อได้ ไม่ตัดฉับเป็น 0 เหมือน ReLU จุดนี้แหละที่สร้างความเนียนในการจำความรู้ชั้นสูง!

🚀 ความลึกซึ้งในความ "สั้นกระชับ"

นักวิจัย AI ไม่ได้ทำโค้ดให้ยาวรุงรัง แต่เขาคัดสรร สูตรคณิตศาสตร์ที่เรียบง่ายที่สุดเพื่อให้รีดสปีดบน GPU ได้สูงสุด

  • การใช้คู่ขนาน gate_out * up_out เสมือนให้เลเยอร์หนึ่งทำหน้าที่ควบคุมวอลลุ่มความเข้มข้นสัญญาณของอีกเลเยอร์หนึ่งโดยตรง
02 · SwiGLU Dimension Pipeline

แผนภาพแสดงการไหลของข้อมูลจริงในฟังก์ชัน forward(x): ขยาย ร่อนสัญญาณผ่าน SiLU คูณผสม แล้วหดกลับเพื่อส่งไปเรียนรู้ต่อใน Layer ถัดไป

📥 Input x [Hidden_Size = 4]
⚡ Gate Projection & SiLU
มิติขยาย: 12 ช่อง (นุ่มนวลสไตล์ Swish)
🚀 Up Projection
มิติขยาย: 12 ช่อง (ขยายพื้นที่คิดเลข)
💥 Expanded State (gate_out * up_out) [Intermediate_Size = 12]
🚀 down_proj(x) Output [Hidden_Size = 4]
03 · Production Code Overview 
# สกัดดึงฟังก์ชันเปิด-ปิดประตูจากคลังแสงภายนอกเข้ามาผูกใช้ตรงๆ
self.act_fn = ACT2FN[config.hidden_act] # รันโหมด "silu" สไตล์ Qwen 3

def forward(self, x):
    # 1. แตกไลน์สายพานขนาน ร่อนสัญญาณผ่าน SiLU
    gate_out = self.act_fn(self.gate_proj(x))
    up_out = self.up_proj(x)

    # 2. บรรจบผสมสัญญาณคูณกันในมิติกว้างสูงสุด
    expanded_state = gate_out * up_out

    # 3. บดอัดมัดรูปทรงกลับเท่าเดิม เพื่อนำไปทำ Residual Add ในสเต็ปถัดไป
    return self.down_proj(expanded_state)

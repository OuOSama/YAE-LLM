01 · หลักการทางคณิตศาสตร์
ทำไม RoPE ไม่ใช่ Absolute Position?

Transformer เดิมบวก positional vector เข้ากับ token embedding ตรงๆ (absolute) ซึ่งทำให้โมเดลจำตำแหน่งแบบสัมบูรณ์ได้ แต่ไม่เก่งเรื่องความสัมพันธ์ระหว่างคำ RoPE เปลี่ยนเป็นการ หมุนเวกเตอร์ในปริภูมิเชิงซ้อน แทน ทำให้โมเดลเข้าใจระยะห่างสัมพัทธ์ (relative distance) ได้โดยตรงจาก dot-product

f(q, m) = q · eimθ
ถอดรหัส: เอาเวกเตอร์คำ q ไปคูณกับ e^(imθ) ซึ่งคือ "ตัวหมุน" ในระนาบเชิงซ้อน — บิดมุมตามตำแหน่ง m ในประโยค เมื่อเอา query กับ key ไป dot กัน ผลลัพธ์จะขึ้นกับ (m - n) เท่านั้น ไม่ใช่ค่า absolute ของแต่ละตัวเลย
สมการ inv_freq
θi = 1 / base(2i/dim)

base = 10,000 (default) หรือ 1,000,000 สำหรับ long context
dim = head_dim (เช่น 128)
i = 0, 2, 4, … (step ทีละ 2 เพราะหมุน 2D คู่)

Rotation Matrix 2×2
R(θ) = [cos θ, −sin θ]
       [sin θ, cos θ]

ทำซ้ำกับทุกคู่ (x₂ᵢ, x₂ᵢ₊₁) ใน head แต่ละคู่มีมุม θᵢ ของตัวเอง เพื่อให้แต่ละ "ช่อง" เรียนรู้รายละเอียดคนละ scale

03 · Production-Ready Design Decisions
Dynamic Calculation

ไม่สร้าง rotation matrix ขนาด seq_len × dim ล่วงหน้า แต่คำนวณ cos/sin เฉพาะตำแหน่งที่ batch นั้นๆ ต้องการจริงๆ ประหยัด VRAM อย่างมากเมื่อ sequence ยาว

Device Agnostic

register_buffer("inv_freq", ...) ทำให้ PyTorch ย้าย tensor ขึ้น GPU โดยอัตโนมัติเมื่อเรียก .to(device) บน module และไม่ผ่าน backprop เพราะไม่ใช่ parameter

Multi-RoPE Support

ตรวจ rope_type ก่อน ถ้าไม่ใช่ "default" จะ fallback ไปดึง ROPE_INIT_FUNCTIONS ของ HuggingFace มาใช้ รองรับ YaRN, Llama3-style scaling โดยไม่ต้อง fork code

Config Robustness

ใช้ getattr(config, "rope_parameters", {}) + or {} และ .get(key, default) ทุกที่ ป้องกัน crash กรณี config ที่ส่งมาไม่มี field นั้นเลย (เช่น ตอน smoke test หรือ deploy)

04 · Linter / Type Fixes จาก Session นี้
Fix 1 · rope_parameters safe access
ปัญหา: config.rope_parameters อาจ return None ถ้าใช้ dataclass หรือ dict ที่ key ไม่ได้ถูก set
แก้: getattr(self.config, "rope_parameters", {}) or {} — double guard ทั้ง missing attribute และ None value
Fix 2 · ROPE_INIT_FUNCTIONS guard
ปัญหา: ถ้า rope_type ไม่อยู่ใน dict จะ KeyError
แก้: เพิ่ม and self.rope_type in ROPE_INIT_FUNCTIONS ก่อน assign — Linter เห็น type narrowing แล้วหยุด warning
Fix 3 · compute_default signature
ปัญหา: config: "YaeConfig" | None = None ทำให้ Pylance ต้องทำ None-check ทุกครั้งที่ใช้ config ภายใน function
แก้: ถอด | None = None ออก — function นี้ require config จริงๆ อยู่แล้ว ทำให้ type ชัดและ linting สะอาด
Fix 4 · Test Suite — Pylance errors
ปัญหา 1: expected_last: Unknown = 1.0 / base — base จาก .get() infer เป็น Unknown
→ แก้: expected_last = 1.0 / float(base)

ปัญหา 2: call compute_default_rope_parameters(config) ไม่ครบ argument
→ แก้: เพิ่ม device=None เป็น keyword arg
